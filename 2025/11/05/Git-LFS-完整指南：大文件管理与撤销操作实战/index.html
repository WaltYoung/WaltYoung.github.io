Git LFS简介

Git LFS（Large File Storage）是Git的一个扩展，用于管理大文件。它用指针替换实际的大文件，将大文件内容存储在远程服务器上。

GitHub免费账户限制：

  • 1GB存储空间
  • 每月1GB带宽

故障排查实战笔记

问题概述

在推送包含大文件的 Git 仓库到 GitHub 时，遇到了多种错误：

  1. 网络连接问题：443端口连接失败、HTTP 408超时、连接重置
  2. LFS 锁定 API 不支持：远程不支持 Git LFS 锁定 API
  3. 文件大小限制：文件超过 GitHub 100MB 限制

根本原因分析

  1. 网络问题：不稳定的网络连接导致推送过程中断
  2. LFS 配置不当：大文件没有被正确追踪或历史记录中仍包含大文件
  3. 文件大小超限：超过 GitHub 的 100MB 单文件限制

错误现象

这个错误通常是由于网络连接不稳定或超时导致的

这个错误表明LFS 锁定 API 不支持

这个错误可能是因为这个文件没有被正确追踪或者历史记录中还有大文件
